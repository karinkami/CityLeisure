## О приложении

**CityLeisure** — веб-сервис для просмотра городских мероприятий и покупки билетов. Интерфейс построен вокруг афиши, карточки события и короткого сценария оформления заказа; отдельно доступен пошаговый **подбор** по предпочтениям и блоки рекомендаций для пользователей с аккаунтом.

**Стек:** Vue 3 · Vite · Pinia · Vue Router · ASP.NET Core 8 · EF Core · PostgreSQL

## Возможности

### 🏠 Главная
- Блок **«Популярное»** — события с наибольшим спросом
- Блок **«Для вас»** — персональные подсказки по избранному, корзине и покупкам (после входа и при наличии активности)
- Быстрые переходы по **категориям** (кино, концерты, театр, бесплатные и др.)

### 🎭 Афиша
- Поиск по названию
- Фильтры: **категория**, **площадка**, **диапазон цены**, **бесплатные**
- Быстрые подборки: **вся афиша**, **популярные**, **рекомендуем** (для авторизованных), **бесплатные**
- Карточки с датой, площадкой, ценой; добавление в **избранное** и **корзину** без ухода со списка
- Отметка событий с **выбором мест** в зале

### 📄 Карточка мероприятия
- Постер, описание, возрастной рейтинг, дата и время
- Информация о **площадке** и ссылка на **карту**
- Покупка билетов: фиксированное количество или **интерактивная схема зала** (ряд и место)
- Блок **похожих событий**

### 🧭 Подбор мероприятий
- Мастер из **4 шагов**: компания → предпочтения (формат, время, ритм, возраст) → фильтры каталога → текстовое уточнение
- Ранжирование на сервере: совпадение с запросом, бюджет, время, популярность продаж
- Список результатов с **пояснением**, почему событие подошло
- Доступен **гостям** и **авторизованным** пользователям

### 🎫 Билеты и заказы
- **Корзина** с изменением количества и удалением позиций
- **Оформление заказа** с контактными данными
- Раздел **«Мои билеты»** — история заказов

### 👤 Аккаунт
- Регистрация и вход
- **Личный кабинет** — профиль пользователя
- **Избранное** — сохранённые мероприятия

### ⚙️ Администрирование
- Панель управления **событиями** (роль Admin): создание и редактирование афиши

## Запуск

**Нужно:** .NET 8 SDK · Node.js (LTS) · PostgreSQL · строка подключения в `backend/CityLeisure.Api/appsettings.json` (`ConnectionStrings:DefaultConnection`)

### Бэкенд (API)

```powershell
cd backend/CityLeisure.Api
dotnet restore
dotnet run
```

- API: **http://localhost:5001**
- Swagger: **http://localhost:5001/swagger** (режим Development)
- Базовый путь REST: **http://localhost:5001/api**

Проверка сборки без запуска: `dotnet build`

### Фронтенд

```powershell
cd frontend
npm install
npm run dev
```

- Приложение: **http://localhost:5173**
- Запросы к API идут на `http://localhost:5001/api` (см. `frontend/src/services/api.js`)

Продакшен-сборка: `npm run build` → папка `frontend/dist/`

> Запускайте **API и фронт в двух терминалах** одновременно.
